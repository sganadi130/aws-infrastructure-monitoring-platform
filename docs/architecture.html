// End-to-End Flow — How Everything Connects

01
Developer
Developer pushes code to GitHub
You modify Terraform files or Lambda Python code on your local machine and run git push origin main. This single action kicks off everything else automatically.
02
CI/CD
GitHub Actions pipeline triggers automatically
GitHub detects the push and runs the terraform.yml workflow. It packages Lambda zip files from Python source, runs terraform init, terraform validate, then terraform apply. AWS credentials come from encrypted GitHub secrets — never from code.
03
IaC
Terraform provisions all AWS infrastructure
Terraform reads the modular code and creates/updates resources in AWS. State is stored remotely in S3 with DynamoDB locking to prevent conflicts. Three modules run: vpc builds networking, compute creates EC2 instances, lambda deploys functions and schedules.
04
Network
VPC isolates and protects all resources
A VPC creates a private network in AWS. Public subnets (with Internet Gateway) host the bastion and monitoring servers. Private subnets (with NAT Gateway) host app servers — reachable only through the bastion, never directly from the internet. Route tables direct traffic to the right gateway.
05
Compute
EC2 instances serve different roles
Bastion host — SSH jump server in public subnet. You SSH into this first, then hop into private instances.
App EC2 x2 — dev servers in private subnets, tagged Environment=dev and ManagedBy=lambda-scheduler so Lambda knows to manage them.
Monitoring EC2 — runs Prometheus + Grafana via Docker Compose.
06
Automation
EventBridge triggers Lambda on a schedule
Three EventBridge cron rules fire automatically:
cron(0 19 ? * MON-FRI *) → 7pm weekdays → triggers ec2_scheduler with {"action": "stop"}
cron(0 8 ? * MON-FRI *) → 8am weekdays → triggers ec2_scheduler with {"action": "start"}
cron(0 0 ? * SUN *) → midnight Sunday → triggers ebs_cleanup
07
Python
Lambda Python functions execute the automation
ec2_scheduler.py uses boto3 to call the AWS EC2 API. It filters instances by tags (Environment=dev + ManagedBy=lambda-scheduler), then stops or starts them depending on the action passed by EventBridge. Results are logged to CloudWatch.

ebs_cleanup.py finds EBS volumes in available state (unattached = wasted money) tagged Environment=dev and deletes them. Logs GB freed per run.
08
Metrics
Prometheus scrapes metrics every 15 seconds
Node Exporter runs on each EC2 instance, exposing system metrics on port 9100. Prometheus visits /metrics on each instance every 15 seconds and stores the data as time-series. Four targets are scraped: both app instances, the monitoring instance itself, and Prometheus scraping itself.
09
Dashboards
Grafana visualises everything in real time
Grafana connects to Prometheus as a data source and renders live dashboards. We imported dashboard 1860 (Node Exporter Full) which shows CPU usage, memory, disk I/O, network traffic, and system load for each instance. Accessible at http://monitoring-ip:3000.

CI/CD Layer
Automated deployment pipeline
GitHub Actions
Push to main
Every git push to main branch triggers the pipeline automatically. No manual steps needed.

Pipeline zips Python files into deployment packages. Zip files are never committed to Git.

🏗
Infrastructure as Code Layer
Modular Terraform with remote state
Terraform
Remote State (S3)
State file lives in S3 — not on your laptop. Versioned, shared, and encrypted.
State Lock (DynamoDB)
Prevents two people running terraform apply simultaneously and corrupting state.
VPC Module
Subnets, IGW, NAT Gateway, route tables. Outputs subnet IDs to compute module.
Compute Module
EC2 instances, security groups. Receives subnet IDs from VPC module as inputs.
Lambda Module
Lambda functions, IAM roles, EventBridge rules. All wired together in one module.
Default Tags
Every resource gets Project, Environment, ManagedBy tags automatically via provider config.
🌐
Network Layer
VPC with public/private subnet separation
AWS VPC
VPC (10.0.0.0/16)
Private network in AWS. Nothing inside is reachable unless explicitly allowed.
Public Subnets x2
10.0.1.0/24 and 10.0.2.0/24. Hosts bastion + monitoring. Has direct internet access via IGW.
Private Subnets x2
10.0.10.0/24 and 10.0.20.0/24. Hosts app servers. No inbound internet. Outbound via NAT.
Internet Gateway
Door between public subnets and internet. Public route table points 0.0.0.0/0 here.
NAT Gateway
Allows private instances to call out to internet (e.g. download updates) without being reachable inbound.
Security Groups
Bastion: SSH from anywhere. App: SSH from bastion only + port 9100 from monitoring only.
🤖
Automation Layer
Serverless cost optimisation
Lambda + EventBridge
EC2 Scheduler
Python boto3. Finds instances by tag. Stops at 7pm, starts at 8am weekdays. ~65% of hours saved.
EBS Cleanup
Finds unattached EBS volumes (status=available) every Sunday and deletes them. Logs GB freed.
IAM Role
Least privilege — only ec2:Stop/Start/Describe and logs:PutLogEvents. Nothing else.
Tag-based targeting
Lambda finds instances by tag not hardcoded IDs. New servers auto-included by adding correct tags.
CloudWatch Logs
Every Lambda execution logged — which instances were stopped/started, errors, execution time.
EventBridge Permissions
Lambda permissions explicitly grant EventBridge the right to invoke each function. Least privilege.
📊
Observability Layer
Real-time metrics and dashboards
Prometheus + Grafana
Node Exporter
Runs on each EC2. Exposes 1000+ system metrics on port 9100. CPU, memory, disk, network.
Prometheus
Scrapes /metrics from all targets every 15s. Stores time-series data. 4 targets: 2 app + monitoring + itself.
Grafana
Connects to Prometheus. Dashboard 1860 (Node Exporter Full) shows live CPU, RAM, disk per instance.
Docker Compose
One command starts all three containers. Volumes persist data across container restarts.
